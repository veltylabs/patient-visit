Manages the full lifecycle of a patient's clinical visit in a healthcare context.
Tracks visit state (FSM), records clinical measurements (vitals), and exposes
HL7/FHIR R4 compliant output through a pure adapter layer.

## Domain Entities

| Entity | Description |
|--------|-------------|
| `MedicalHistory` | A single patient visit episode (status, diagnosis, prescription, doctor/patient snapshots) |
| `MeasurementType` | Reference catalogue of vital measurement types (weight, blood pressure, etc.) with LOINC/UCUM codes |
| `ClinicalMeasurement` | A recorded vital measurement taken during a visit |
| `HistoryDetail` | Clinical catalog items applied during the visit (linked to external Catalog) |

## Visit Lifecycle (FSM)

```
created → arrived → triaged → in_progress → completed
   └──────────────────────────────┴──► cancelled
```

## Design Notes

- **Soft references:** `patient_id`, `doctor_id`, `reservation_id`, and `catalog_item_id`
  are string soft-refs — no foreign key constraints between domains.
- **Snapshots:** Patient and doctor names are captured at visit creation time and never
  fetched cross-module at read time.
- **No REST API:** FHIR output is provided by pure adapter functions, not endpoints.
