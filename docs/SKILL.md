# clinical-encounter — LLM Skill Summary

## Purpose
Manages the full lifecycle of a patient's clinical visit: from scheduling to discharge.
Tracks visit state via FSM, records clinical measurements (vitals), and exposes FHIR R4
JSON on demand through a pure adapter layer.

## Key Files
| File | Role |
|------|------|
| `model.go` | Domain entities: `MedicalHistory`, `MeasurementType`, `ClinicalMeasurement`, `HistoryDetail`; FSM constants and transition table |
| `model_orm.go` | Auto-generated ORM helpers — DO NOT EDIT. Regenerate with `ormc` after model changes |
| `fhir_types.go` | FHIR R4 DTO structs (no business logic) |
| `fhir_adapter.go` | Pure functions translating internal → FHIR; no DB access |
| `mcp.go` | `Module` struct, `New(db, pub)` constructor |
| `mcp_visit.go` | MCP tools for `MedicalHistory` CRUD + FSM transitions |
| `mcp_visit_status.go` | Tool: `update_visit_status` (FSM enforcement) |
| `mcp_measurement.go` | Tool: `add_clinical_measurement` |
| `mcp_measurement_type.go` | Tool: `list_measurement_types` |
| `mcp_detail.go` | Tool: `add_history_detail` |
| `tests/setup_test.go` | Shared DB setup for all `!wasm` tests |

## Core Constraints
- `MedicalHistory.ID` is a string PK (unixid). All other entity IDs also use unixid.
- `Status` is enforced by `visitTransitions` FSM. Invalid transitions return an error.
- Snapshots (`PatientNameSnapshot`, `DoctorNameSnapshot`, etc.) are set at creation.
  Never fetched cross-module at read time.
- `EventPublisher` interface is injected; concrete implementation lives outside this module.
- Module uses `tinywasm/orm` for all DB access. Tests use `tinywasm/sqlite` `:memory:`.
- `model_orm.go` is generated by `ormc`. Run `ormc` after any change to `model.go`.

## FSM: Visit Lifecycle
```
created → (mark_arrived) → arrived → (mark_triaged) → triaged → (start_visit) → in_progress → (complete) → completed
   │                           │                           │
   └──(cancel)──► cancelled ◄──┴───────────────────────────┘
```

## FHIR Adapter
`fhir_adapter.go` exposes pure conversion functions:
- `MedicalHistoryToFHIR(h *MedicalHistory) FHIREncounter`
- `MeasurementToFHIRObservation(m *ClinicalMeasurement, mt *MeasurementType) FHIRObservation`
No REST endpoints. Call these functions wherever FHIR output is needed.

## MCP Registration
```go
m, _ := clinicalencounter.New(db, eventPublisher)
m.RegisterTools(srv) // srv is *mcp.MCPServer
```
